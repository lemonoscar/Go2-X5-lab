# P4: Flat Arm Unlock On Top Of P1

## 1. 定位

这个 `P4` 不是 `train_route/step_04_arm_fixed_base_imitation.md` 里那条固定底座 imitation-only 分支。

这里的 `P4` 是当前 `Go2-X5` route 的实用延续版本：

- 起点：`P1` 的 flat foundation checkpoint
- 目标：在 **平地** 上把机械臂从“锁死默认位姿”过渡到“可控解锁”
- 约束：保持整套 policy 的输入输出维度不变，保证可以直接 `resume`

对应新任务：

- `RobotLab-Isaac-Velocity-Flat-Go2-X5-ArmUnlock-v0`

## 2. 续训模型

直接续训用这个 checkpoint：

- `logs/rsl_rl/go2_x5_foundation_flat/2026-03-12_12-11-27/model_8000_armdims.pt`

如果手里只有旧的 `model_8000.pt`，要先迁移成 `model_8000_armdims.pt` 再训。

## 3. 输入输出维度

`P1` 和 `P4` 保持完全一致的 route 接口：

- actor obs: `259`
- critic obs: `259`

分解如下：

- `base_lin_vel`: `3`
- `base_ang_vel`: `3`
- `projected_gravity`: `3`
- `velocity_commands`: `3`
- `joint_pos`: `18`

所以 `P1 -> P4` 可以直接续训，不需要改网络首尾层。

## 4. P4 环境语义

`P4` 的核心不是加 rough，而是先在 flat 上把 arm 问题单独解决掉。

- terrain: flat
- `num_envs = 2048`
- standing ratio: `1.0`
- base command:
  - `lin_vel_x = (0.0, 0.0)`
  - `lin_vel_y = (0.0, 0.0)`
  - `ang_vel_z = (0.0, 0.0)`
- base command 不再继续 curriculum 放大
- arm command 使用 `use_default_offset=True`
- 当前默认 arm 姿态按仓库里的最新资产设置，基准是 `0 0 0 0 0 0`
- 训练目标更偏向 `stand-still arm control`，而不是低速 locomotion

## 5. Arm 解锁流程

`P4` 里不是直接把 arm 命令一次性开到大范围，而是做 range curriculum。

初始 arm command range:

- `joint1: (-0.20, 0.20)`
- `joint2: (-0.35, 0.35)`
- `joint3: (-0.35, 0.35)`
- `joint4: (-0.20, 0.20)`
- `joint5: (-0.18, 0.18)`
- `joint6: (-0.18, 0.18)`

目标 arm command range:

- `joint1: (-1.20, 1.20)`
- `joint2: (-1.20, 1.20)`
- `joint3: (-1.20, 1.20)`
- `joint4: (-0.80, 0.80)`
- `joint5: (-0.70, 0.70)`
- `joint6: (-0.70, 0.70)`

range curriculum:

- `curriculum_iterations = 128`

arm action scale 也同步放大到：

- `joint1: 0.80`
- `joint2: 1.00`
- `joint3: 1.00`
- `joint4: 0.60`
- `joint5: 0.50`
- `joint6: 0.50`

## 6. 奖励函数

`P4` 使用 reward curriculum，从 `P1` 权重平滑过渡到 `P4 target`，避免从锁臂 checkpoint 直接跳到开臂任务。

reward curriculum:

- `curriculum_iterations = 128`

### 6.1 P4 target reward weights

- `track_lin_vel_xy_exp = 4.0`
- `track_ang_vel_z_exp = 2.0`
- `lin_vel_z_l2 = -1.5`
- `ang_vel_xy_l2 = -0.08`
- `flat_orientation_l2 = -0.45`
- `base_height_l2 = -0.2`
- `body_lin_acc_l2 = -0.015`
- `joint_torques_l2 = -1.5e-5`
- `joint_acc_l2 = -1.0e-7`
- `joint_pos_limits = -2.0`
- `joint_power = -1.0e-5`
- `stand_still = -3.0`
- `joint_pos_penalty = -1.0`
- `action_rate_l2 = -0.01`
- `undesired_contacts = -1.0`
- `contact_forces = -1.0e-4`
- `feet_air_time = 0.0`
- `feet_air_time_variance = 0.0`
- `feet_contact_without_cmd = 0.3`
- `feet_slide = -0.12`
- `feet_gait = 0.0`
- `arm_joint_pos_tracking_l2 = -3.0`
- `arm_joint_vel_l2 = -0.001`
- `arm_joint_acc_l2 = -7.5e-7`
- `arm_joint_torques_l2 = -6.0e-5`
- `arm_action_rate_l2 = -0.010`
- `arm_joint_pos_limits = -2.0`
- `arm_joint_deviation_l2 = 0.0`
- `arm_motion_tilt_penalty = -0.25`
- `arm_action_in_unstable_base = -0.08`
- `arm_stable_track_bonus = 1e-6`

### 6.2 为什么这样设

- `arm_joint_deviation_l2` 直接放到 `0.0`，避免 arm 在大范围跟踪时还被默认位姿强拉回去。
- `arm_joint_pos_tracking_l2` 保留中等强度，确保 arm 真正跟 command 走。
- `arm_motion_tilt_penalty` 和 `arm_action_in_unstable_base` 明确约束“arm 不能把底盘打崩”。
- `stand_still / joint_pos_penalty / flat_orientation / ang_vel_xy` 全部加重，目标是 arm 动的时候底盘尽量别走。
- `feet_air_time / feet_gait` 归零，避免 P4 继续鼓励步态行为。
- `arm_stable_track_bonus` 现在基本关闭，因为当前实现的 gate 还是按绝对目标范数，不适合在 `use_default_offset=True` 下强开。

## 10. 训练中的检查点

建议重点看 4 个现象：

- arm command 非零时，`arm_joint_pos_tracking_l2` 是否持续下降
- `flat_orientation_l2` 和 `lin_vel_z_l2` 是否明显恶化
- standing 环境里是否出现 arm 高频抖动
- base 有轻微移动命令时，arm 是否还能跟踪

如果 `1000 ~ 1500` 轮内 arm 仍然几乎不动，优先检查：

- 默认姿态是不是你期望的全零姿态
- arm command range curriculum 是否生效
- checkpoint 用的是不是 `model_8000_armdims.pt`

## 11. P4 结束后的去向

`P4` 结束后有两条合理路线：

- 去 `P3` 风格的 rough arm 阶段：把 flat 上学到的 arm/base 协调迁到 rough
- 去严格的 `train_route step_04/05` 分支：单独做 fixed-base 或 standing-base manipulation

如果你当前最关心的是“机器人在平地上先把 arm 用起来”，那先做这个 `P4` 是对的。
